# Linux Basics for Servers

A desktop Linux user mostly clicks. A server admin mostly types. This section turns the terminal from a scary black window into your most powerful tool.

1. [The Shell & Files](/linux-basics/shell-and-files)
2. [Permissions & Ownership](/linux-basics/permissions)
3. [Users, Root & sudo](/linux-basics/users-and-sudo)
4. [Package Management (dnf)](/linux-basics/package-management)
5. [Processes & systemd](/linux-basics/processes-and-systemd)

> **Mental model** — A Linux server is just a long-running program (the kernel) that gives other programs (services) access to files, network, and CPU. Almost everything you do as an admin is one of three things: **start/stop a service**, **change a config file**, or **read a log**. Every lesson in this section feeds into one of those.
